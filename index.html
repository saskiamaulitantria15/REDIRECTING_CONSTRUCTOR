Sesi 11: Redirecting Constructors

Review: Factory Constructors

  • Format Penulisan: Menggunakan kata kunci factory Class.name().
  • Karakteristik Utama: Tidak harus selalu membuat instance baru. Dapat mengembalikan objek yang sudah ada di memori atau objek dari kelas turunannya (subclass).
  • Skenario Penggunaan: Umumnya diterapkan untuk pembuatan pola singleton, mekanisme penyimpanan berbasis caching, atau pengelolaan kumpulan objek (object pools).

Pengantar Redirecting Constructors

  • Definisi: Sebuah constructor yang tidak menginisialisasi objeknya secara mandiri, melainkan langsung "mengalihkan" atau meneruskan parameter panggilan ke constructor lain di dalam kelas yang sama.
  • Tujuan Utama: Meminimalkan penulisan kode yang berulang demi mematuhi prinsip DRY (Don't Repeat Yourself) agar struktur kode menjadi lebih bersih dan mudah dirawat.
  • Aturan Sintaksis: Pengalihan dilakukan dengan menuliskan : this(...) pada bagian initializer list (setelah nama constructor dan tanda titik dua).
  • Karakteristik Tubuh Kode: Constructor jenis ini tidak boleh memiliki body atau tidak memiliki blok tanda kurung kurawal { }.

Sintaks Dasar Redirecting Constructor

  • Struktur Kelas: Memiliki variabel utama x dan y yang bersifat final.
  • Constructor Utama: Berperan sebagai target pengalihan yang menerima parameter lengkap Point(this.x, this.y).
  • Metode Pengalihan:
    • Point.origin() dialihkan langsung menuju koordinat pusat : this(0, 0).
    • Point.onXAxis(double x) dialihkan hanya dengan sumbu X : this(x, 0).
    • Point.onYAxis(double y) dialihkan hanya dengan sumbu Y : this(0, y).
  • Eksekusi Utama (Main): Saat objek-objek tersebut dipanggil di fungsi utama (void main), sistem otomatis mengarahkan nilai bawaan tersebut ke constructor utama secara ringkas.
    • //Nama: Saskia Mauli Tantria
      Nim: 251410034
      Kelas: SI2A
      
  class Point {
  final double x;
  final double y;

  // Konstruktor utama
  Point(this.x, this.y) {
        // print('Point created at ($x, $y)');
  }

  // Konstruktor pengarah (redirecting constructor) - mengarah ke konstruktor utama
  Point.origin() : this(0, 0);

  Point.onXAxis(double x) : this(x, 0);

  Point.onYAxis(double y) : this(0, y);

  @override
  String toString() {
    return 'Point($x, $y)';
  }
}

void main() {
  // Penggunaan
  var origin = Point.origin(); // Point(0, 0)
  var pointOnX = Point.onXAxis(5); // Point(5, 0)
  var pointOnY = Point.onYAxis(3); // Point(0, 3)

  // Cetak variabel origin, pointOnX, dan pointOnY
  print('Origin: $origin');
  print('Point on X-axis: $pointOnX');
  print('Point on Y-axis: $pointOnY');
}

Masalah Tanpa Redirecting Constructor

  • Kendala Variabel Final: Variabel bertipe final (seperti width dan height) wajib diinisialisasi langsung saat objek dibuat dan nilainya tidak bisa diubah di dalam body { }.
  • Kesalahan Duplikasi Logika (Error): Upaya mengisi nilai variabel final di dalam tubuh constructor (seperti contoh width = side; atau width = 100;) akan menghasilkan ERROR: final field.
  • Solusi Masalah: Untuk menghindari eror modifikasi variabel final tersebut, kode harus ditulis ulang menggunakan initializer list tradisional atau dialihkan secara bersih memakai redirecting constructor.
    • //Nama: Saskia Mauli Tantria
      Nim: 251410034
      Kelas: SI2A

  class Rectangle {
  final double width;
  final double height;
  
  Rectangle(this.width, this.height);
  
  // Tanpa redirecting - duplikasi logika
  Rectangle.square(double side) {
    width = side;    // ERROR: final field
    height = side;
  }
  
  Rectangle.withDefault() {
    width = 100;     // ERROR: final field
    height = 50;
  }
}

Solusi dengan Redirecting Constructor

  • Struktur Solusi: Daripada mengisi nilai di dalam body { } yang memicu eror, nilai langsung dioper ke constructor utama menggunakan sintaksis : this(...).
  • Penerapan Kasus:
    • Rectangle.square(double side) dialihkan menjadi persegi dengan mengirim parameter yang sama : this(side, side).
    • Rectangle.withDefault() dialihkan untuk membuat ukuran bawaan : this(100, 50).
    • Rectangle.fromWidth(double width) dialihkan menggunakan rumus matematika tertentu (misalnya Golden Ratio) : this(width, width * 0.618).
    • //Nama: Saskia Mauli Tantria
      Nim: 251410034
      Kelas: SI2A

class Rectangle {
  final double width;
  final double height;
  
  // Main constructor
  Rectangle(this.width, this.height) {
    print('Rectangle $width x $height created');
  }
  
  // Redirecting constructor
  Rectangle.square(double side) : this(side, side);
  
  Rectangle.withDefault() : this(100, 50);
  
  Rectangle.fromWidth(double width) : this(width, width * 0.618); // Golden ratio

  @override
  String toString() {
    return 'Rectangle(width: ${width.toStringAsFixed(2)}, height: ${height.toStringAsFixed(2)})';
  }
}

void main(){
// Penggunaan
var square = Rectangle.square(10);      // 10x10
var defaultRect = Rectangle.withDefault(); // 100x50
var goldenRect = Rectangle.fromWidth(100); // 100x61.8

print('Square: $square');
print('Default Rectangle: $defaultRect');
print('Golden Ratio Rectangle: $goldenRect');
}

Redirecting Constructor dengan Named Parameters

  • Karakteristik Target: Constructor utama (Person) menggunakan parameter bernama dengan kata kunci required untuk validasi input wajib.
  • Aturan Pengalihan: Saat mengalihkan, nama parameter wajib ditulis secara eksplisit di dalam this(...).
  • Penerapan Nilai Default:
    • Person.anonymous() mengalihkan nama ke 'Anonymous' dan usia ke 0 via : this(name: 'Anonymous', age: 0).
    • Person.adult(String name) mengunci nilai usia langsung ke angka 18 via : this(name: name, age: 18).
    • Person.child(String name, int years) meneruskan variabel lokal 'years' ke parameter 'age' via : this(name: name, age: years, email: null).

      Nim: 251410034
      Kelas: SI2A

      //Nama: Saskia Mauli Tantria
//Nim: 251410034
//Kelas: SI2A

class Person {
  final String name;
  final int age;
  final String? email;

  // Main constructor dengan named parameters
  Person({
    required this.name,
    required this.age,
    this.email,
  });

  // Redirecting dengan default values
  Person.anonymous() : this(name: 'Anonymous', age: 0);

  Person.fromName(String name) : this(name: name, age: 0);

  Person.adult(String name) : this(name: name, age: 18);

  Person.child(String name, int years) : this(name: name, age: years, email: null);

  @override
  String toString() {
    return 'Name: $name, Age: $age, Email: ${email ?? "N/A"}';
  }
}

void main() {
  print('Demonstrating Person class constructors:\n');

  // Using the main constructor with required named parameters
  var john = Person(name: 'John Doe', age: 30, email: 'john.doe@example.com');
  print('1. Main constructor (John): ${john.toString()}');

  // Using the main constructor without an optional parameter
  var jane = Person(name: 'Jane Smith', age: 25);
  print('2. Main constructor (Jane, no email): ${jane.toString()}');

  // Using Person.anonymous()
  var anonymous = Person.anonymous();
  print('3. Person.anonymous(): ${anonymous.toString()}');

  // Using Person.fromName(String name)
  var bob = Person.fromName('Bob');
  print('4. Person.fromName(\'Bob\'): ${bob.toString()}');

  // Using Person.adult(String name)
  var alice = Person.adult('Alice');
  print('5. Person.adult(\'Alice\'): ${alice.toString()}');

  // Using Person.child(String name, int years)
  var charlie = Person.child('Charlie', 7);
  print('6. Person.child(\'Charlie\', 7): ${charlie.toString()}');

  var diana = Person.child('Diana', 12);
  print('7. Person.child(\'Diana\', 12): ${diana.toString()}');
}

Praktik 1 – Configuration Class

  • Fungsi Utama: Memusatkan semua pengaturan koneksi basis data (seperti host, port, username, password, database, dan status SSL) ke satu constructor utama.
  • Pengalihan Profil Lingkungan (Environment):
    • DatabaseConfig.development() otomatis mengalihkan konfigurasi ke server lokal pengembangan dengan kredensial bawaan (localhost, user pengembang, dan SSL nonaktif).
    • DatabaseConfig.production() mengalihkan konfigurasi ke server produksi dengan keamanan ketat (menggunakan domain asli, kredensial produksi, dan mewajibkan SSL aktif).
    • DatabaseConfig.localhost() menyederhanakan panggilan khusus pengujian lokal tanpa perlu menuliskan detail parameter berulang kali.
  • Pengalihan Parameter Sebagian: Menyediakan constructor fleksibel yang hanya meminta nama host dan nama basis data saja, sementara nilai parameter lainnya diisi oleh nilai default sistem melalui pengalihan.

      Kelas: SI2A

class DatabaseConfig {
  final String host;
  final int port;
  final String username;
  final String password;
  final String database;
  final bool useSSL;

  // Konstruktor utama
  DatabaseConfig({
    required this.host,
    required this.port,
    required this.username,
    required this.password,
    required this.database,
    this.useSSL = false,
  });

  // Konstruktor redirecting untuk lingkungan berbeda
  DatabaseConfig.development()
      : this(
          host: 'localhost',
          port: 5432,
          username: 'dev_user',
          password: 'dev_pass',
          database: 'dev_db',
          useSSL: false,
        );

  DatabaseConfig.production()
      : this(
          host: 'db.example.com',
          port: 5432,
          username: 'prod_user',
          password: 'prod_pass',
          database: 'prod_db',
          useSSL: true,
        );

  DatabaseConfig.localhost()
      : this(
          host: 'localhost',
          port: 3306,
          username: 'root',
          password: '',
          database: 'test',
        );

  // Konstruktor redirecting dengan sebagian parameter
  DatabaseConfig.withHost(String host, String database)
      : this(
          host: host,
          port: 5432,
          username: 'default',
          password: 'password',
          database: database,
        );

  @override
  String toString() {
    return 'DatabaseConfig(\n'
        '  host: $host,\n'
        '  port: $port,\n'
        '  username: $username,\n'
        '  password: ${'*' * password.length},\n' // Masker kata sandi untuk tampilan
        '  database: $database,\n'
        '  useSSL: $useSSL\n'
        ')';
  }
}

void main() {
  print('--- Konfigurasi Database ---');

  // 1. Menggunakan konstruktor utama
  final customConfig = DatabaseConfig(
    host: 'my.custom.server',
    port: 1234,
    username: 'admin',
    password: 'secure_password',
    database: 'important_data',
    useSSL: true,
  );
  print('\n[Konfigurasi Kustom]');
  print(customConfig);

  // 2. Menggunakan konstruktor development
  final devConfig = DatabaseConfig.development();
  print('\n[Konfigurasi Development]');
  print(devConfig);

  // 3. Menggunakan konstruktor production
  final prodConfig = DatabaseConfig.production();
  print('\n[Konfigurasi Production]');
  print(prodConfig);

  // 4. Menggunakan konstruktor localhost
  final localConfig = DatabaseConfig.localhost();
  print('\n[Konfigurasi Localhost]');
  print(localConfig);

  // 5. Menggunakan konstruktor withHost
  final specificHostConfig =
      DatabaseConfig.withHost('another.server.net', 'my_app_db');
  print('\n[Konfigurasi dengan host dan database spesifik]');
  print(specificHostConfig);

  print('\n--- Akhir Konfigurasi ---');
}

Redirecting Constructor dengan Validasi

  • Constructor Utama Privat: Memanfaatkan constructor utama yang bersifat privat (BankAccount._(...)) agar pembuatan objek tidak bisa dilakukan secara sembarangan dari luar kelas.
  • Penggabungan dengan Factory: Menggunakan factory BankAccount.create(...) untuk menjembatani proses validasi sebelum objek benar-benar dibuat.

Multiple Redirecting Levels

  • Struktur Dasar Objek: Kelas Product memiliki banyak properti wajib seperti id, name, price, category, stock, hingga tanggal kedaluwarsa expiryDate.
  • Tingkatan Pengalihan (Redirection Levels):
    • Level 1 Redirecting (Basic): Constructor hanya meminta data paling esensial (id, name, price). Properti lainnya diisi otomatis dengan nilai default standar (kategori 'General' dan stok 0) lalu dialihkan ke constructor utama.
    • Level 2 Redirecting (With Category): Memberikan tingkat fleksibilitas lebih dengan mengizinkan input kategori khusus dari pengguna, sementara nilai stok tetap dikunci otomatis ke angka 0 sebelum dialihkan.
    • Level 3 Redirecting (Full Custom): Mengizinkan pengisian seluruh parameter secara kustom termasuk tanggal kedaluwarsa opsional, yang kemudian diteruskan secara utuh ke penampung utama.

Redirecting Constructor vs Factory Constructor

  • Tujuan: Redirecting memanggil constructor lain di class yang sama, sedangkan Factory menangani logic kompleks, caching, atau subclass.
  • Return: Redirecting harus mengembalikan instance class tersebut, sedangkan Factory bisa mengembalikan instance yang sudah ada atau subclass.
  • Body: Redirecting tidak memiliki body (hanya ekspresi : this(...)), sedangkan Factory memiliki body lengkap berisi logika.
  • Final fields: Redirecting mendukung proses inisialisasi variabel final, sedangkan Factory tidak bisa langsung menginisialisasi di dalam body.
  • Use case: Redirecting untuk default parameters dan convenience, sedangkan Factory untuk singleton, object pool, dan polymorphism.

Praktik 2 – API Response dengan Redirecting

  • Fokus Kasus: Memetakan dan mengelola berbagai status respons dari server (seperti respons sukses, respons eror/gagal, atau data kosong) ke dalam sebuah kelas objek tunggal (ApiResponse).
  • Tujuan Latihan: Menyederhanakan penulisan status aplikasi dengan cara menyediakan constructor pengalihan khusus untuk setiap skenario respons (seperti success, error, notFound, dan unauthorized), sehingga kode di sisi antarmuka menjadi lebih bersih saat membaca data.

Redirecting Constructor dengan Initializer List

  • Pengalihan Parameter Tambahan: Mengisi otomatis properti bawaan seperti ID unik menggunakan timestamp milidetik, nama pengguna bawaan, dan status aktif false.
  • Pengalihan dengan Kalkulasi Nilai: Memproses string input terlebih dahulu (misalnya mengubah nama menjadi huruf kecil semua atau merangkai string email baru) sebelum dilempar ke constructor utama.
  • Validasi di dalam Tubuh Kode: Menunjukkan bahwa jika constructor pengalihan membutuhkan pemeriksaan logika yang kompleks, validasi tersebut harus dilakukan sebelum tanda pengalihan selesai dieksekusi agar tidak memicu eror pada variabel final.

Best Practices Redirecting Constructor

  • Gunakan untuk: Default parameter values, convenience constructors, and alternative ways to create objects.
  • Hindari untuk: Complex logic (gunakan factory), Caching (gunakan factory), dan Subclass creation (gunakan factory).
  • Tips: Jaga agar rantai pengalihan tetap pendek, dokumentasikan tujuan setiap constructor, dan gunakan nama yang bermakna pada named constructor.

Ringkasan

  • Ringkasan:
    1. Sintaksis penulisan wajib menggunakan : this(...) tanpa body.
    2. Untuk memanggil constructor lain di class yang sama.
    3. Mengurangi duplikasi kode.
    4. Tidak bisa berisi logic kompleks.
